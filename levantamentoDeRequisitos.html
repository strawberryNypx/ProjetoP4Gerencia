Levantamento de Requisitos

Casos de Uso:

Reservar Quarto
Descrição: Funcionário registra reserva para cliente
Check-in
Descrição: O cliente ocupa o quarto reservado
Check-out
Descrição: Processo de saída do cliente e cálculo de pagamento
Registrar Consumo
Descrição: Adição de consumos (bebidas, refeições etc.)
Gerar Relatórios
Descrição: Visualização de relatórios por período, ocupação e receita
Cadastrar Funcionário
Descrição: Administrador registra dados dos funcionários
Gerenciar Quartos
Descrição: Atualização de informações dos quartos (preço, tipo, status)

Requisitos Funcionais (RF):

RF01
O sistema deve permitir o cadastro de clientes.
RF02
O sistema deve permitir o cadastro de funcionários.
RF03
O sistema deve permitir o cadastro e gerenciamento dos quartos.
RF04
O sistema deve registrar reservas de quartos.
RF05
O sistema deve permitir o check-in de clientes.
RF06
O sistema deve registrar o check-out e calcular o valor da hospedagem.

Requisitos Não Funcionais (RNF):

RNF01
O sistema deve ser acessado apenas por usuários autenticados.
RNF02
A interface deve ser simples e intuitiva para uso em recepção.
RNF03
O sistema deve garantir integridade e consistência dos dados.
RNF04
O tempo de resposta para operações de cadastro e reserva deve ser inferior a 3 segundos.
RNF05
O sistema deve armazenar os dados em um banco de dados relacional (MySQL ou similar).
RNF06
O sistema deve ser compatível com navegadores modernos (Google Chrome, Firefox)
